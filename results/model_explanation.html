💡 建模思想

我们不是简单地把数据丢进神经网络——而是把"足球分析师"的思维编码成特征和模型结构。

核心思路：将"专家知识"编码为特征

一个足球分析师判断比赛胜负时，会考虑：

① 球队绝对实力

这支球队有多强？→ Elo 评分系统，根据每场比赛结果动态更新

② 近期状态

最近几场踢得怎么样？→ 滑动窗口统计：近 10 场胜率、进球、失球

③ 对战历史

两队以前交手谁占优？→ Head-to-Head 特征：历史交锋胜/平/负

④ 比赛背景

是世界杯决赛还是友谊赛？中立场地还是主场？→ 赛事权重 + 场地标记

⑤ 球队风格交互

有些球队天生克制另一些球队 → 球队 Embedding + Self-Attention，让模型自主学习球队间的"化学效应"

Elo 评分：球队能力的"通用货币"

Elo 系统最初用于国际象棋，后被广泛用于足球排名。核心公式： 

新 Elo = 旧 Elo + K × Importance × GoalFactor × (实际结果 − 预期结果)

预期胜率 = 1 / (1 + 10−(Elo差 + 主场加成) / 400)

世界杯权重 2.0x，预选赛 1.5x；净胜 3 球以上额外加权；主场加成 +100 Elo 分。 每场比赛后两队 Elo 同时更新，形成一个随时间演化的能力评估系统。

网络结构细节

模块配置作用
Team Embedding 327 × 64 维，正态初始化 σ=0.05 将每支球队映射到 64 维向量空间，相似的球队向量接近。额外加上 Home/Away Indicator（可学习参数）区分主客场角色。
Team Attention 4 头 Multi-Head Attention + FFN 让主客两队的 Embedding 互相"注意"——例如巴西 vs 阿根廷会激活不同于巴西 vs 弱旅的交互模式。这是传统机器学习做不到的。
Match Encoder 21 → 128 → 128（2 层 GELU） 将手工特征映射到高维空间，与 Embedding 维度对齐。
ResBlock × 3 [256, 256, 128]，Dropout 0.25 DenseNet 风格残差块（BatchNorm → GELU → Linear），带 Shortcut 连接，防止梯度消失。
分类头 128 → 64 → 32 → 3 输出 3 个 logit：客胜 / 平局 / 主胜，经 Softmax 得概率。
辅助进球头 128 → 64 → 2 同时预测主客队进球数（Softplus 激活），作为辅助任务提升主任务效果。

损失函数：Focal Loss

FL(pt) = −αt × (1 − pt)γ × log(pt)
    α = [1.17, 1.43, 0.69]   ← 类别权重（平衡少数类）
    γ = 2.0                    ← 聚焦参数（压易样本，抬难样本）

总损失 = FocalLoss(分类) + 0.15 × MSE(进球预测)

为什么用 Focal Loss 而非交叉熵？因为平局只占 22.7%，普通 CE Loss 会让模型忽略平局。 Focal Loss 自动降低"容易分类"的样本（如西班牙 vs 弱旅）的梯度，迫使模型关注难例。 辅以进球预测的多任务学习，进一步提供监督信号。

🔑 关键洞察与经验

1. 时间序列数据不可随机打乱

如果随机切分训练/测试集，模型准确率可以轻松达到 65%+——但这毫无意义。 因为模型会"看到"2024 年的比赛来预测 2010 年。我们严格用过去预测未来， 这才是真实场景中的有效模型。

2. Elo 是最好的单特征

在消融实验中，仅用 Elo 差值的简单逻辑回归就能达到 ~50% 准确率。 所有 21 维特征加在一起，贡献了剩余 ~7 个百分点的提升。 Elo 之所以有效，是因为它天然融合了对手强度和时序演化。

3. NaN 数据是隐形杀手

数据集中包含 2026 年世界杯的未来比赛，这些比赛的比分为 NaN。 我们第一次运行 Elo 计算时，NaN > NaN 返回 False，所有未来比赛被当作 0-0 平局处理， 导致 Elo 被系统性拉向均值。修复后所有球队 Elo 才恢复正常。

4. 多任务学习有效

同时预测比分（辅助任务）帮助模型学到了关于"球队攻防强度"的更丰富表示。 去掉进球预测头后，F1 下降约 2 个百分点。

5. Draw is football

平局是足球预测中最难的部分。即使世界顶级博彩模型，平局准确率也仅在 25–30%。 我们的模型在平局上的 Recall 为 31%，已属 competitive。淘汰赛中我们退化为 Elo 比较来处理平局，这是务实的选择。
